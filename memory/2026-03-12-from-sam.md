## OpenClaw Inline Button System Analysis
- Studied OpenClaw's `/models` command implementation for inline button patterns
- **Button format**: `message(action="send", buttons=[[{text, callback_data, style?}]])`
- **Callback patterns**: Uses prefixes like `mdl_prov` (providers), `mdl_list_` (list), `mdl_sel_` (select)
- **Telegram config**: Requires `capabilities.inlineButtons` set to "dm", "group", "all", or "allowlist"
- **Implementation approach**: Multi-step selection flow with chained questions
- **Adaptation for alerts**: Can use pattern `alert_<section>_<choice>` (e.g., `alert_title_1`, `alert_severity_info`, `alert_recipient_hub`)
- **Key insight**: OpenClaw uses 2-column layout for providers, groups by count, handles pagination
